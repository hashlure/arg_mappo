# ARG-MAPPO：时空注意力驱动的内在动机算法框架
一款为多智能体强化学习（MARL）深度定制的算法框架。该框架以 MAPPO 为主干，融合 AREL（智能体-时间注意力）核心思想，创新性构建了基于**“全局惊奇度分配”** 的内在动机（好奇心）探索机制。为保障底层数学严谨性与工程训练的极致稳定性，本代码在纯注意力分配之外，增设三层物理防御装甲，彻底解决多智能体好奇心探索中的一系列致命难题。

## 🧠 核心架构：从物理观测到好奇心驱动的底层拆解
算法信息流并非简单堆砌，而是遵循严格的张量降维与重构逻辑。整个 learn 过程可拆解为以下四个递进阶段：

### 阶段一：🔍 惊奇度的产生（Curiosity Generation）
- **组件**：GlobalRewardPredictor（$B\_Net$）
- **底层逻辑**：如何定义“未知”？网络 B 试图基于当前状态 $s$ 和联合动作 $u$ 预测环境真实奖励 $r$。当环境出现网络 B 无法预测的动态（预测误差极大）时，将其定义为**“局部惊奇度”（Local Surprise）**。
- **计算目标**：收集整条轨迹的预测误差，求和得到回合总惊奇度 $E_{total}$。

### 阶段二：🌌 时空重构与信用分配（Spatiotemporal Redistribution）
- **组件**：Time_Agent_Transformer（$ARG\_Net$）
- **底层逻辑**：总惊奇度已确定，但究竟是哪一时刻、哪个智能体的动作导致了这份惊奇？
- **特征提炼**：通过 MLP 将冰冷的原始高维物理观测 $E$ 投影至富含语义的隐空间。
- **纵向扫描（Temporal Attention）**：加入因果掩码（Causality Mask），寻找时间序列上的“转折点”。
- **横向检索（Agent Attention）**：在同一时间截面上，计算智能体间的相互作用强度，定位“关键智能体”。
- **置换不变坍缩（Deep Sets）**：将融合历史与同伴特征的 $Z$ 矩阵，经求和抹平智能体顺序，最终坍缩为每个时间步的预测内在奖励 $\hat{r}_t$。
- **双重约束优化**：
  - 回归对齐：$\sum \hat{r}_t \approx E_{total}$（总分必须匹配）；
  - 方差正则：$\omega \cdot Var(\hat{r}_t)$（拒绝“大锅饭”，强制拉开得分差距）。

### 阶段三：🛡️ 三重底层防御装甲（The Triple Defense System）
直接将好奇心注入 PPO 具有灾难性风险。我们在信息流融合前（$r_{mixed} = r + \beta * e_{predict}$）布置三道关卡：
| 防御机制 | 实现逻辑 | 核心作用 |
|----------|----------|----------|
| 防“吵闹的电视机”：奖励缓冲区门控（RewardBufferGate） | 维护包含最近 320 局得分的滑动队列作为“及格线” | 若智能体为追求惊奇度乱操作（环境得分低于均值），则直接物理截断其好奇心奖励（乘 0），仅放行“有价值突破”的惊奇度 |
| 防“PPO 价值评估剥离”：动态 Batch 标准化 | 对通过门控的内在奖励做局部 $\mu=0, \sigma=1$ 标准化，并硬性截断极值 | 避免方差极高的超大好奇心奖励瞬间撑爆 PPO 的 Critic 网络，保证 Advantage 计算不受污染 |
| 防“冷启动灾难”：非线性高斯预热（Gaussian Warm-up） | 好奇心权重 $\beta$ 挂载随时间增长的系数 $1 - \exp(-(t/T)^2)$ | 网络权重随机的混沌初期压制噪声，待网络具备基础评估能力后，平滑释放探索权限 |

### 阶段四：⚔️ 策略演化（PPO Optimization）
经过上述层层处理，平滑、稠密且指向明确的融合奖励 $r_{mixed}$ 被送入标准 MAPPO 主干，通过 GAE 优势计算与 PPO 截断更新，完成策略的最终进化。

## 📂 核心代码结构映射
| 文件/类名 | 物理意义与职责 |
|-----------|----------------|
| mappo.py → MAPPO | 算法主脑：统筹 Actor-Critic 优化与 ARG 机制的生命周期 |
| mappo.py → RewardBufferGate | 安检员：维护得分基线，生成物理截断掩码 |
| arg_net.py → Time_Agent_Transformer | 时空解构器：执行 $Q \cdot K^T$ 寻优，输出高质量信用分配蓝图 |
| b_net.py → GlobalRewardPredictor | 好奇心源泉：被 Target 网络平滑追踪，产生预测误差 |
| runner.py → worker_process | 平行采样器：多进程无锁采样，突破 MARL 探索效率瓶颈 |

## 🛠️ 数据维度流动备忘录
阅读/修改 Time_Agent_Transformer 时，需牢记张量的物理视角转换：
| 维度格式 | 物理含义 |
|----------|----------|
| [B, N, T, D] | 上帝视角的原始物理观测（批次、智能体数、时间步、特征维度） |
| [B*N, T, D] | 抽离时间线：每个智能体独立审视自身历史（时间注意力） |
| [B*T, N, D] | 切出横截面：所有智能体在同一帧交互（智能体注意力） |
| [B, T, 1] | 坍缩标量化：抹平个体差异，输出每一时间步的团队探索奖励 |
